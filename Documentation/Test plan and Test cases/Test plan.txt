## 2. Test Strategy: Process-Oriented Approach

### 2.1 Test Levels and Types

| **Test Level** | **Description** | **Focus** |
|----------------|-----------------|------------|
| **Unit Testing** | Testing individual components (functions, classes) of the client and server code. | Code correctness, internal logic. |
| **Integration Testing** | Testing interfaces/interactions between modules (client-server, DB, APIs). | Data flow, API contract adherence. |
| **System Testing** | End-to-end testing of the fully integrated application. | Functional & non-functional requirements. |
| **User Acceptance Testing (UAT)** | Final testing by end-users/stakeholders. | Usability, business process validation. |

#### Test Types
- **Functional Testing:** Verify features (Authentication, Trading, Quizzes, Leaderboard).  
- **Security Testing:** Validate secure handling of credentials and data protection.  
- **Performance Testing:** Measure responsiveness and stability under load.  
- **Usability Testing:** Assess user interface intuitiveness and navigation.  

---

### 2.2 Detailed Testing Processes and Steps

#### **Process 1: User Authentication and Profile Management**

| **Process Step** | **Description** | **Key Test Focus** |
|------------------|-----------------|--------------------|
| **P1.1: Registration** | User provides email and password to create an account. | Data validation, unique email, account creation success. |
| **P1.2: Login** | User provides credentials to access account. | Credential validation, JWT token, dashboard redirection. |
| **P1.3: Logout** | User terminates the session. | Token invalidation, secure logout, redirect to login. |
| **P1.4: Profile Update** | User updates personal info or password. | Data persistence, password security, immediate updates. |

---

#### **Process 2: Paper Trading Simulation**

| **Process Step** | **Description** | **Key Test Focus** |
|------------------|-----------------|--------------------|
| **P2.1: Stock Search** | User searches for a stock by ticker. | API connectivity (Alpha Vantage), data retrieval, invalid ticker handling. |
| **P2.2: Buy Transaction** | User buys a stock. | Balance check, portfolio update, transaction log. |
| **P2.3: Sell Transaction** | User sells an owned stock. | Ownership validation, balance update, transaction log. |
| **P2.4: Portfolio View** | User views current holdings and balance. | Accurate calculations, correct display. |
| **P2.5: History Tracking** | User views past transactions. | Complete, accurate record of trades. |

---

#### **Process 3: Gamification and Leaderboard**

| **Process Step** | **Description** | **Key Test Focus** |
|------------------|-----------------|--------------------|
| **P3.1: Quiz Attempt** | User starts and answers quiz. | Random questions, answer submission, correct display. |
| **P3.2: Reward Earning** | User earns currency for correct answers. | Reward calculation, wallet update. |
| **P3.3: Leaderboard Update** | Userâ€™s score is ranked. | Score logic, ranking accuracy, real-time updates. |
